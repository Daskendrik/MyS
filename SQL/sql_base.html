Таблица. Свойства

В общем, ничего нового о таблице особо не напишешь. Она состоит из строк и столбцов, имет свое уникальное название. Можно немного поговорить об особенностях таблиц, с которыми работает SQL. 

  1. Поле это пересечение столбца и строки
  2. Каждая таблица должна содержать в себе уникальный столбец - ключ (на нашем проекте это row_id). При создании записи в таблицу можно заполнять это поле вручную, но чаще всего это поле ставится на автоматическое заполнение. 
  3.  NULL и "" в полях - это разные вещи

Для этого необходимо пройти 2 этапа: создание таблицы и ее заполнение.

 Создание таблицы 

У нас будут следующие колонки:  Row_id, Family, LastName, First_name, Middle_name, Telephone, Mark, Model. Название таблицы Telephone. Поля, которые не должны быть пустыми должны маркироваться NOT NULL. В таблицах так же должен присутствовать ключ, в нашей таблице это row_id, и чтобы самим не запоминать и не придумывать этот ключ поручим это таблицы и подпишем AUTO_INCREMENT.  Так же зпишем PRIMARY KEY, об этом ппж.

CREATE TABLE Telephone (

Row_id int AUTO_INCREMENT,

 Family int NOT NULL,

LastName varchar(20) NOT NULL,

First_name varchar(20) NOT NULL,

 Middle_name varchar(20),

Telephone int NOT NULL,

Mark varchar(20),

Model  varchar(20)),

PRIMARY KEY (Row_id)

 Заполнение таблицы

Теперь необходимо ее заполнить.

Для заполнения используем INSERT INTO ... VALUES

INSERT INTO имя_таблицы VALUES ('значение_первого_столбца','значение_второго_столбца', ..., 'значение_последнего_столбца');

или

INSERT INTO имя_таблицы ('имя_столбца', 'имя_столбца') VALUES ('значение_первого_столбца','значение_второго_столбца');

Если необходимо внести несколько строк, то после VALUES каждая строчка будет в ().

INSERT INTO имя_таблицы ('имя_столбца', 'имя_столбца') VALUES ('значение_первого_столбца','значение_второго_столбца');

('значение_первого_столбца','значение_второго_столбца');
