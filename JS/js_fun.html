Встроенные функции и операторы

Фкнции это тоже операторы, но для себя их все таки отделяю.

alert ДОПИСАТЬ О ТОМ, КАК ПИСАТЬ ЗНАЧЕНИЯ,СТРОКИ,ВСЕ ВМЕСТЕ, ПЕРЕНОС СТРОК!

Этот код отобразит окно в браузере и приостановит дальнейшее выполнение скриптов до тех пор, пока пользователь не нажмёт кнопку «OK». От себя добавлю, что скобки должны быть впритык к слову алерт! Шаблончик такой:

alert(message);

Это небольшое окно с сообщением называется модальным окном. Понятие модальное означает, что пользователь не может взаимодействовать с интерфейсом остальной части страницы, нажимать на другие кнопки и т.д. до тех пор, пока взаимодействует с окном. В данном случае – пока не будет нажата кнопка «OK». Например:

alert("Хай пипл!")

prompt

Этот код отобразит модальное окно с текстом, полем для ввода текста и кнопками OK/Отмена. Шаблончик такой:

result = prompt(title, [default]);

title это текст-вопрос
default - какой ответ по дефолту, он не обязатедлен, но лучше всегда ставить.
Если пользователь нажимает "Отмена", введенное значение засчитывается как null.
Получается, что значение переменной присваевает пользователь.

let age = prompt('Сколько тебе лет?', 100);
alert(`Тебе ${age} лет!`); // Тебе 100 лет!

confirm

Функция confirm отображает модальное окно с текстом вопроса question и двумя кнопками: OK и Отмена.
Результат true, если нажата кнопка OK. В других случаях – false.
Шаблончик:

result = confirm(question);

Получается, что значение переменной присваевает пользователь, в данном случае, это true или false.

let isBoss = confirm("Дашка главная?");
alert( isBoss ); // true, если нажата OK

typeof

Этот оператор возвращает тип аргумента. Тип проверка:

let a = 15
alert (typeof a) //нам выведет, что это число

Функции

Функции являются основными «строительными блоками» программы. Сущестует 2 вида объявления функций: Function Declaration и Function Expression.

Function Declaration (Объявление Функции)

Конструкция функции выглядит так:

function имя(параметры) {
  ...тело...
}

Переменные, объявленные внутри функции, видны только внутри этой функции. Мы можем передать внутрь функции любую информацию, используя параметры (также называемые аргументами функции).

//Функция выглядит так
function showMessage(from, text) {
  alert(from + ': ' + text);
}
по кнопке мы вызываем так: showMessage('Дашка', 'Привет незнакомец!')

Если в функции что то делается с аргументом, то эти изменения за пределы функции не уйдут.

Параметры по умолчанию

нужны в том случае, если мы не передаем параметр. При чем мы можем передавать как значение, так и вызывать другую функцию. Например:

function showMessage(from, text = anotherFunction()) { // или text = "текст не добавлен"
// anotherFunction() выполнится только если не передан text
// результатом будет значение text
}

Возврат значения

Функция может вернуть результат, который будет передан в вызвавший её код. Например:

function sum(a, b) {
  return a + b;
}
let result = sum(1, 2);
alert( result ); // 3

Директива return может находиться в любом месте тела функции. Как только выполнение доходит до этого места, функция останавливается, изначение возвращается в вызвавший её код (присваивается переменной result выше). Пример можно глянуть в примерах, номер 10.

Как подобрать имя функции

Название функции - глагол+предмет, над которым совершается действие.

Function Expression (Функциональное Выражение)

Не знаю как точно описать, но получчается, что мы создаем переменную и ей присваеваем новосозданную функцию. Например, создаим переменную и функцию:

let sayHi = function() {
  alert( "Привет" );
};

Если после имени функции нет круглых скобок, то мы НЕ вызываем ее.Для понимания вот пример, мы получим строки кода.

function sayHi() {
  alert( "Привет" );
}
alert( sayHi ); // выведет код функции

Важный пример!

function sayHi() { // (1) создаём
alert( "Привет" );
}
let func = sayHi; // (2) копируем
func(); // Привет // (3) вызываем копию (работает)!
sayHi(); // Привет // прежняя тоже работает (почему бы нет)

Объявление Function Declaration (1) создало функцию и присвоило её значение переменной с именем sayHi.
В строке (2) мы скопировали её значение в переменную func. Обратите внимание (ещё раз): нет круглых скобок после sayHi. Если бы они были, то выражение func = sayHi() записало бы результат вызова sayHi() в переменную func, а не саму функцию sayHi.
Теперь функция может быть вызвана с помощью обеих переменных sayHi() и func().

Функциями-колбэки или просто колбэки

Ключевая идея в том, что мы передаём функцию и ожидаем, что она вызовется обратно (от англ. «call back» – обратный вызов) когда-нибудь позже, если это будет необходимо. Пример такой функции урок 12.

Функции стрелки

Довольно инетересно. Самый коротки синтаксис создания функции - функции стрелки. Шаблон:

let func = (arg1, arg2, ...argN) => expression

Такой код создаёт функцию func с аргументами arg1..argN и вычисляет expression с правой стороны с их использованием, возвращая результат. Если перевести на прошлый шаблон, то это выглядеть будет так:

let func = function(arg1, arg2, ...argN) {
  return expression;
};

Например

let sum = (a, b) => a + b;

То есть, (a, b) => a + b задаёт функцию с двумя аргументами a и b, которая при запуске вычисляет выражение справа a + b и возвращает его результат.
Стоит еще вот что запомнить:

let sum = (a, b) => { // фигурная скобка, открывающая тело многострочной функции
let result = a + b;
return result; // при фигурных скобках для возврата значения нужно явно вызвать return
};

В данном случае тело функции большое, поэтому используются скобки, но в таком случае необходимо писать, что вовращать!
И еще: если у нас только один аргумент, то круглые скобки вокруг параметров можно опустить и если нет аргументов, указываются пустые круглые скобки.

Функция-конструктор

Функции-конструкторы являются обычными функциями. Но есть два соглашения:

  • Имя функции-конструктора должно начинаться с большой буквы.
  • Функция-конструктор должна вызываться при помощи оператора "new".
