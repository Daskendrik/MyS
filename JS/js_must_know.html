То, что стоит запомнить

Тут я хочу написать то, что стоит помнить и повторять.

  1. Если код маленький, то можно оставить в html его. Но если кода много, лучше переносить его в отдельный фаил. Тогда Браузер один раз запомнит код в кеше и не придется ему заново грузить, даже если мы используем этот же JS на другой странице.
  2. Нельзя указывать путь к JS и одновременно его прописывать в html.
  3. Последовательность выполнения скриптов такая, какая написана на странице.
  4. Имена переменных не должны начинаться с цифры, может содержать буквы, цифры, _ и $. Лучше латинские буквы.
  5. Регистр имеет значение
  6. Есть зарезервированные слова, которые нельзя использовать для создания переменной
  7. Переменной можно задать значение константы
  8. Дополнительная переменная – это добро, а не зло
  9. JS – динамически типизированный – тип данных есть, но не привязывается к переменной.
  10. Скрипты никогда не останавливаются с фатально ошибкой, мы можем получить только NaN.
  11. Типа данных char тут нет, значения типа @ % это все строки.
  12. Сложение преобразует все в строчку (если хотя бы один из операндов является строкой, то второй будет также преобразован к строке.)
  13. Вычитание, умножение и деление преобразует все в числа
  14. Операнд (аргументы)– то, к чему применяется оператор
  15. Унарным называется оператор, который применяется к одному операнду. (например, -1 – меняет знак на противоположный)
  16. Бинарным называется оператор, который применяется к двум операндам. (например, 2 - 1)
  17. Преобразование типа • Number(...) тоже самое, что и +
  18. Инкремент/декремент можно применять только на переменных
  19. Результат сравнения имеет логический тип
  20. Для сравнения строк используется алфавит. Например А это 1, Б это 2, значит Б больше А. Если в строке несколько букв, то буквы сравниваются поочереди, пока одна из них не окажется больше/меньше другой.
  21. При сравнении значений разных типов JavaScript приводит каждое из них к числу.
  22. Значения null и undefined равны == друг другу и не равны любому другому значению.
  23. Переменный, объявленные в цикле, живут только в нем!
  24. Директива continue позволяет избегать вложенности
  25. Нельзя использовать break/continue справа от оператора „?“
  26. Вызов break/continue возможен только внутри цикла, и метка должна находиться где-то выше этой директивы.
  27. Чтобы организовать бесконечный цикл, используют конструкцию while (true).
  28. Возможно использовать return и без значения. Это приведёт к немедленному выходу из функции.
  29. Переменные, объявленные внутри функции, видны только внутри этой функции. Функция обладает полным доступом к внешним переменным и может изменять их значение. Если одноимённая переменная объявляется внутри функции, тогда она перекрывает внешнюю. Желательно сводить использование глобальных переменных к минимуму.
  30. Возможно использовать return и без значения. Это приведёт к немедленному выходу из функции.
  31. Одна функция – одно действие
