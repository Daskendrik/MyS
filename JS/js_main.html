Подключение

Есть 2 способа подключить скрипт к странице. Первый, он же самый элементарный, написать его в head в блок < script> < / script >. Второй вариант, он же в большинстве случаев лучший вариант, это вынести скрипты в отдельный фаил и подключить его так:

< script type="text/javascript" src="js/main.js"> < / script >

Тогда Браузер один раз запомнит код в кеше и не придется ему заново грузить скрипт, даже если мы используем этот же JS на другой странице.

Строение кода

Инструкции – команды. Их может быть множество. Они могу перечислять ентр или точкой с запятой. Так же, они могут заключаться в функцию.

alert ("Всем приве!")
alert ("меня зовут Дашка")

Объявление переменных можно перечислять тоже несколькими способами. Через ентр или запятую

let a, b
let f = 1,
c = 15

Переменные и константы

Переменные раньше задавлись var, но теперь появилось слово let. Например

let a = 45

Есть константы, то есть ее нельзя будет изменять в дальнейшем. Обычно константы пишут заглавными буквами и с нижним подчеркиванием, если значение известно до начала сценария.

const COLOR_RED = "#F00";
const pageLoadTime = /* время, потраченное на загрузку веб-страницы */;

Типы данных

Как хорошо, что в JS нет привязки к типам походу. Переменная может содержать и число, потом буквы, потом опять число. Всего 8 типов данных:

  1. Число (number). Это как целые, так и с плавающей точкой. Так же есть специальные числовые значения: Infinity (бесконечность), NaN (вычислительная ошибка).
  2. BigInt. Числа не могут содержать числа больше 2 в 53 степени, или меньше -2 в 53 степени. Для таких чисел есть тип BigInt. Для обозначения такого числа, в конце необходимо написать n, например, 123456789123456789123456789n.
  3. Строка (String). Строки заключаются в кавычки – ‘Дашка’. Кавычки есть 3 типов: ‘’, “”, ``. Первые два вида простые, а третий вид имеет расширенную функциональность. В такие ‘’ кавычки можно вписать выражение (пример в уроке х).
  4. Булевой (логический) тип. Правда или ложь.
  5. Значение undefined. Тоже только одно значение, обозначающее, что значение не было присвоено. Обычно используется для проверок.
  6. Объекты. Тип object используются для хранения коллекций данных.
  7. Символы. Тип symbol используется для создания уникальных идентификаторов объектив.

Оператор typeof. Этот оператор возвращает тип аргумента. Тип проверка:

let a = 15
alert (typeof a) //нам выведет, что это число

Операторы

If

Оператор if(...) вычисляет условие в скобках и, если результат true, то выполняет блок кода. Результат условия преобразуется в логический тип.
Стоит помнить: Число 0, пустая строка "", null, undefined и NaN становятся false. Из-за этого их называют «ложными» («falsy») значениями. Остальные значения становятся true, поэтому их называют «правдивыми» («truthy»).
Стандартный шаблон выглядит так

if (условие без кавычек) {
выполняемый код, если условие выполняется;
}

If else

Когда нам нужно сделать или одно или другое, в зависимости от выполнения условий, мы добавляем блок else. Такой шаблон выглядит так:

if (условие без кавычек) {
выполняемый код, если условие выполняется;
} else {
выполняемый код, если условие не выполняется;
}

Else if

Если у нас множество значений для условий и к каждому свой код, можно использовать вместо тысячи if конструкцию else if. Такой шаблон выглядит так:

if (условие 1) {
выполняемый код, если условие выполняется;
} else if (условие 2) {
выполняемый код, если условие 2 выполняется;
}
} else if (условие 3) {
выполняемый код, если условие 3 выполняется;
} else {
выполняемый код, если ни одно условие не выполнилось;
}

Оператор ?

Дада, ? - оператор в JS. Он служит для того, чтобы задать переменной значение в зависимости от условия. По сути, это сокращенный if else в некоторых случаях. Шаблончик вот такой (ниогда скобки у условий убирают):

let a = условие ? значение1 : значение2
//Это тоже самое что и вот это
if (условие1) {
a = значение1;
} else {
a = значение2;
}

Примерчик для понимания

let a;
let age = prompt('Сколько вам лет?', '');
if (age>20) {
a = 'взрослый';
} else {
a = 'мелкий';
}
// теперь используем ?
let b = (age>20) ? 'взрослый' : 'мелкий';
alert(a);
alert(b);

Так же можно использовать несколько ? с несколькими условиями

let a = условие1 ? значение1 :
условие2 ? значение2:
условие3 ? значение3:
значение4

Иногда всю конструкцию if заменяют ? (пример можно посотреть на странице примеров, тест номер 4) Но использовать так оператор ? не рекомендуют.

Логический оператор || или

Традиционно в программировании ИЛИ предназначено только для манипулирования булевыми значениями: в случае, если какой-либо из аргументов true, он вернёт true, в противоположной ситуации возвращается false. Если значение не логического типа, то оно к нему приводится в целях вычислений. Пример:

let age = 15
if (age < 14 || age > 100 || age == 15) {
alert('Вам 15, алкоголь нельзя!')
}

Интересное применение показано в примере, тест 5.

Логический оператор И &&

В традиционном программировании И возвращает true, если оба аргумента истинны, а иначе – false. Данный оператор при появлении первого false останавливается. Все как и с ИЛИ, только возвращает первое ложное. Если все значения верны, возвращает последнее.

Оператор ! (не)

Сначала оператор приводит аргумент к логическому значению, затем возвращает противоположное. Иногда двойное !! исползуется для преобразования значения к логическому типу.

Циклы

Циклы используются для повторения одного и тоже действия несколько раз

While

Цикл while выполняется, пока условие(condition) истина. Одно выполнение тела цикла по-научному называется итерация. Шаблон такого цикла выглядит следующим образом:

while (condition) {
код, тело цикла
}

Любое выражение или переменная может быть условием цикла, а не только сравнение: условие while вычисляется и преобразуется в логическое значение. Например, while (i) – более краткий вариант while (i != 0):

let i = 3;
while (i) {
alert(i);
i--;
}

Do while

Цикл сначала выполнит тело, а затем проверит условие condition, и пока его значение равно true, он будет выполняться снова и снова. Тело цикла будет выполнено всегда хотя бы один раз. Шаблончик выглядит так:

do{
тело цикла
} while (condition);

For

Цикл For является довольно сложным, но чаще используют именно его. Шаблон следующий:

for (начало;условие;шаг){
тело цикла
}

Пример:

for(let i = 0; i < 3; i++) {
alert(i);
}

Начало i = 0 - выполняется один раз в цикле
Условие проверяется перед каждой итерацией цикла
i++ шаг выполняется на каждой итерации перед проверкой условия
Тело - и так ясно =)
Переменный, объявленные в цикле, живут только в нем!
Если переменная была объявлена, то первая часть цикла (начало) можно оставить пустым:

let i = 0; for(; i < 3; i++){
alert(i)
}

Прерывание цикла break

Иногда нам необходимо выйти из цикла, как только мы получим необходимый результат(например), для этого можно использовать прерывание цикла. Например, пользователь вводит числа, а потом решает перестать, и чтобы цикл завершился, мы добавим в код прерывание:

let sum = 0;
while(true){
let value = +prompt("Введите число", '');
if(!value) break;
sum+=value;
}
alert ('Сумма: ' + sum);

Вообще, сочетание «бесконечный цикл + break» – отличная штука для тех ситуаций, когда условие, по которому нужно прерваться, находится не в начале или конце цикла, а посередине.

Переход к следующей итерации: continue

Директива continue – «облегчённая версия» break. При её выполнении цикл не прерывается, а переходит к следующей итерации (если условие все ещё равно true).

Выводим только нечетные значения

for(let i = 0; i < 10; i++){
if(i % 2 ==0) continue;
alert (i);
}

Меткия для break/continue

Если необходимо выйти сразу из нескольких циклов, то используются метки. Метка имеет вид идентификатора с двоеточием перед циклом:

lableName: fpr(...) {
...
}

Рассмтрим пример. Пользователю необходимо ввести значения для координат от (0,0) до (2,2). Но если мы жмем Отмена или не вводим ничего, то тогда необходимо выйти сразу из 2 циклов.

outer: for (let i = 0; i < 3; i++) {
for (let j = 0; j < 3; j++) {
let input = prompt ('Значение на координатах (${i}),{j}', '');
if(!input) break outer;
}
}
alert('Готово');

Вызов break/continue возможен только внутри цикла, и метка должна находиться где-то выше этой директивы.

Конструкция "switch"

Конструкция switch заменяет собой сразу несколько if. Переменная x проверяется на строгое равенство первому значению value1, затем второму value2 и так далее. Если соответствие установлено – switch начинает выполняться от соответствующей директивы case и далее, до ближайшего break (или до конца switch). Если ни один case не совпал – выполняется (если есть) вариант default. Конструкция выглядит таким образом:

switch(x) {
case 'значение х1':
  ...
  [break]
case 'значение х2':
  ...
  [break]
default:
  ...
  [break]
}

Если break нет, то выполнение пойдёт ниже по следующим case, при этом остальные проверки игнорируются.

Интересный пример, switch и case допускают любое выражение в качестве аргумента:

let a = "1";
let b = 0;
switch (+a) {
  case b + 1:
    alert("Выполнится, т.к. значением +a будет 1, что в точности равно b+1");
    break;
  default:
    alert("Это не выполнится");
}

Группирока case

Иногда, чтобы не писать один и тот же код для нескольких вариантов кейсов, можно их сгруппировать. Сгруппируем 2 и 3 значение.:

switch(x) {
case 'значение х1':
  ...
  [break]
case 'значение х2':
case 'значение х3':
  ...
  [break]
default:
  ...
  [break]
}

Функции

Функции являются основными «строительными блоками» программы. Сущестует 2 вида объявления функций: Function Declaration и Function Expression.

Function Declaration (Объявление Функции)

Конструкция функции выглядит так:

function имя(параметры) {
  ...тело...
}

Переменные, объявленные внутри функции, видны только внутри этой функции. Мы можем передать внутрь функции любую информацию, используя параметры (также называемые аргументами функции).

//Функция выглядит так
function showMessage(from, text) {
  alert(from + ': ' + text);
}
по кнопке мы вызываем так: showMessage('Дашка', 'Привет незнакомец!')

Если в функции что то делается с аргументом, то эти изменения за пределы функции не уйдут.

Параметры по умолчанию

нужны в том случае, если мы не передаем параметр. При чем мы можем передавать как значение, так и вызывать другую функцию. Например:

function showMessage(from, text = anotherFunction()) { // или text = "текст не добавлен"
// anotherFunction() выполнится только если не передан text
// результатом будет значение text
}

Возврат значения

Функция может вернуть результат, который будет передан в вызвавший её код. Например:

function sum(a, b) {
  return a + b;
}
let result = sum(1, 2);
alert( result ); // 3

Директива return может находиться в любом месте тела функции. Как только выполнение доходит до этого места, функция останавливается, изначение возвращается в вызвавший её код (присваивается переменной result выше). Пример можно глянуть в примерах, номер 10.

Как подобрать имя функции

Название функции - глагол+предмет, надо которым совершается действие.

Function Expression (Функциональное Выражение)

Не знаю как точно описать, но получчается, что мы создаем переменную и ей присваеваем новосозданную функцию. Например, создаим переменную и функцию:

let sayHi = function() {
  alert( "Привет" );
};

Если после имени функции нет круглых скобок, то мы НЕ вызываем ее.Для понимания вот пример, мы получим строки кода.

function sayHi() {
  alert( "Привет" );
}
alert( sayHi ); // выведет код функции
